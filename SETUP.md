# manga-cdc Setup Guide

Generated by the configure wizard. Follow these steps to get started.

## Prerequisites

- Docker & Docker Compose
- Go 1.26+ (optional, for running the scraper on the host)
- Java 21+ (optional, for running the notification service on the host)

## Environment

1. Copy the environment file:
   ```bash
   cp .env.example .env
   ```
2. Fill in the values in `.env`.

## Local Development (Docker Compose)

Postgres and Redpanda are included. The scraper publishes chapter events directly to Redpanda.

```bash
docker compose up --build -d
```

```bash
docker compose logs scraper
```

## Verification

```bash
docker compose ps
curl http://localhost:2112/metrics | grep scraper_
curl http://localhost:8080/actuator/health
```

## Troubleshooting

- Scraper skips Kafka publish when `KAFKA_BROKERS` is empty.
- Notification service health may report `DOWN` when Kafka or database credentials are missing.

## Adding Manga Sources

Implement the `SourceAdapter` interface in `scraper/internal/adapter/`.
